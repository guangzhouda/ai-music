# 小说成歌方案设计

## 1. 问题定义

用户希望导入一整篇小说、章节或角色设定后，系统能够“理解内容”并生成歌曲。  
这里不能依赖“大模型一次性阅读全文”，因为长文本在以下方面不稳定：

- 上下文长度受限
- 成本不可控
- 召回不稳定
- 对角色、片段、场景的细粒度生成不友好

因此本项目采用 `导入全文 + 切块索引 + 检索增强 + 结构化提示词` 的方案。

## 2. 处理链路

### 2.1 导入

输入来源：

- 整篇小说
- 某一章
- 某个角色设定
- 一段剧情片段

### 2.2 文本清洗

处理：

- 去除空行和噪声
- 标准化段落
- 保留章节结构

### 2.3 分块

本项目首版采用段落聚合切块：

- 每 3 个自然段组成一个 chunk
- 若文本较短，则保留单 chunk

每个 chunk 保存：

- `label`
- `text`
- `tokens`
- `vector`

### 2.4 向量化

当前仓库为了保证开箱可跑，内置了轻量本地向量化策略：

- 文本分词
- 哈希投影
- 固定维度向量
- 余弦相似度检索

这不是最终生产形态，但足够支撑：

- 全文导入
- 多模式检索
- 角色/场景/节选召回

后续升级方向：

- OpenAI Embeddings
- 火山引擎 Embedding
- 本地 `bge` / `m3e` / `gte` 系列模型
- PostgreSQL `pgvector`

## 3. 五种生成模式

### 3.1 全文成歌

输入：

- 小说全文
- 风格
- 聚焦主旨

输出：

- 作品主题歌

### 3.2 节选成歌

输入：

- 指定片段
- 风格
- 当前情绪

输出：

- 面向高潮或片段的歌曲版本

### 3.3 角色主题曲

输入：

- 角色名
- 角色描述
- 人物弧光

输出：

- 针对单角色的人物歌

### 3.4 场景配乐

输入：

- 场景描述
- 节奏要求
- 情绪要求

输出：

- 场景驱动配乐或弱歌词歌曲

### 3.5 风格重编

输入：

- 同一故事素材
- 不同风格规则

输出：

- 多版本风格实验

## 4. 提示词构造方法

在生成歌曲前，系统会将以下信息拼装成结构化 prompt：

- 文档标题
- 模式类型
- 聚焦内容
- 检索出的相关 chunk
- 文档摘要
- 关键词
- 角色线索
- 风格规则

这也是当前后端 `composeNovelPrompt` 的职责。

## 5. 为什么这是适合当前项目的方法

原因有三点：

1. 不依赖一次性长上下文  
2. 可扩展到角色、场景、节选等细分功能  
3. 可以逐步替换成更强的真实向量模型而不重写业务层

## 6. 当前仓库实现状态

已经具备：

- 文本导入接口
- 自动摘要
- 自动关键词抽取
- chunk 构建
- 本地向量化
- 相似度检索
- 小说成歌 prompt 组装

后续增强建议：

- 真正的章节切分

- 角色关系图谱

- 多文档知识库
