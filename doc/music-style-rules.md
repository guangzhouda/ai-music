# 音乐制作规则与风格规则整理

本文档整理的是“网上已有的创作规则、平台规范和常见风格约束”，用于给 AI 音乐生成提供稳定的提示词模板，而不是把音乐创作误写成僵硬标准。

## 1. 平台与工程层规则

### 1.1 Loudness 与母带分发

主流流媒体平台都会做响度归一化，因此“无限压大音量”不是正确策略。  
对 AI 音乐项目来说，这意味着提示词与后处理应该优先保证：

- 人声清晰
- 低频不糊
- 副歌动态明显
- 不要为了响度牺牲层次

参考：

- [Spotify Loudness Normalization](https://support.spotify.com/us/artists/article/loudness-normalization/)
- [Apple Digital Masters](https://www.apple.com/apple-music/apple-digital-masters/)

### 1.2 结构优先于堆叠元素

电子流行、流行叙事、影视化歌曲都高度依赖结构分段，而不是把所有乐器从头铺到尾。  
对于生成式音乐提示词，建议明确写出：

- `intro`
- `verse`
- `pre-chorus`
- `chorus`
- `bridge`
- `outro`

这样比只写“来一首好听的歌”稳定得多。

### 1.3 Hook 必须可记忆

流行歌曲最重要的仍然是可重复、可识别的 Hook。  
AI 提示词里应优先限制：

- 副歌短句长度
- 重复次数
- 关键词
- 高潮情绪词

## 2. 常见风格规则

### 2.1 华语流行电影感

适合：

- 一键成歌
- 都市夜景
- 成长、告别、追梦题材

常见特征：

- `78-96 BPM`
- 主歌留白
- 副歌拉宽空间感
- 钢琴 + Pad + 弦乐 + 稳定鼓组

提示词要点：

- 明确叙事情绪
- 副歌需要一句可记忆主旨
- 编曲不要覆盖主唱

### 2.2 国风抒情

适合：

- 仙侠
- 古代宿命
- 角色主题曲

常见特征：

- `70-88 BPM`
- 民族乐器作为辨识点
- 主旋律线条更婉转
- 常见意象：月、雪、长街、归途、故梦

提示词要点：

- 不要只写“古风”，要补充情绪和角色关系
- 指明主要民族乐器
- 副歌仍要有现代流行可记忆性

### 2.3 电子流行

适合：

- 城市、赛博、年轻、速度感
- 高识别度单曲

常见特征：

- `100-124 BPM`
- 合成器 Hook
- Sidechain 呼吸感
- 主歌克制，Drop 或副歌释放

提示词要点：

- 写清节奏密度
- 说明主导音色
- Hook 要短，节拍要明确

### 2.4 摇滚主题曲

适合：

- 热血成长
- 群像叙事
- 战斗或反抗题材

常见特征：

- `110-148 BPM`
- 失真吉他与真实鼓组
- 副歌靠节奏和群唱推进
- 尾奏适合重复主题动机

提示词要点：

- 强化节奏型
- 吉他不要抢主唱频段
- 副歌必须可喊唱

## 3. 小说成歌的专用规则

### 3.1 全文成歌

目标是提炼“作品主旨”，不是复述全部剧情。  
提示词重点：

- 故事主主题
- 主要人物关系
- 总体情绪线
- 最终落点

### 3.2 节选成歌

目标是抓住一个片段的情绪峰值。  
提示词重点：

- 场景地点
- 当前冲突
- 情绪转折
- 是否适合做副歌高潮

### 3.3 角色主题曲

目标是围绕人物身份与命运。  
提示词重点：

- 角色标签
- 欲望与缺陷
- 成长弧
- 外在形象和内在情绪反差

### 3.4 场景配乐

目标是让音乐服务镜头或章节。  
提示词重点：

- 节奏密度
- 配器空间
- 是否以氛围为主
- 是否弱化歌词

## 4. 本项目可直接使用的提示词模板

### 4.1 一键成歌模板

```text
写一首 {风格} 歌曲，主题是 {主题}，情绪为 {情绪}，速度 {BPM 区间}，主歌克制，副歌爆发，核心乐器为 {乐器}，副歌需要明确 Hook。
```

### 4.2 角色主题曲模板

```text
围绕角色 {角色名} 创作主题曲。人物关键词：{关键词}。人物弧光：{成长线}。希望歌曲表达 {情绪}，风格 {风格}，副歌突出人物命运感和记忆点。
```

### 4.3 场景配乐模板

```text
为场景 {场景描述} 创作音乐。当前冲突是 {冲突}，节奏密度 {密度}，需要 {情绪}，配器以 {配器} 为主，避免过多人声干扰叙事。
```

## 5. 对本项目的落地结论

工程上不应只保存“风格名”，而应保存一整套规则：

- `BPM`
- `情绪`
- `配器`
- `编排说明`
- `适用场景`

因此本项目已经把风格规则写进共享配置，并同步给：

- 前端下拉菜单
- 小说成歌提示词构造
- 文档规则库
